Firebase Android SDK

Let’s start with the Android SDK, one of my personal favorites.

A bunch of libraries from the Android SDK were updated. Most notably:

Authentication got a KTX library

General Availability of the KTX libraries was announced by the end of March 2020 and recently, Authentication got its KTX library.

This means that if you use Kotlin to build your Android Apps, you can update your code to be more concise and idiomatic while leveraging some of the Kotlin language features.
To get started with Authentication KTX, simply add the -ktx suffix to the gradle dependency and make sure you’re using version 19.3.1 or above:

dependencies {
  // Other dependencies
  implementation "com.google.firebase:firebase-auth-ktx:19.3.1"
}

This allows you to use some of the new Extension Functions:

You can find more Extensions included in the Auth KTX on the reference docs.

Firebase Crashlytics SDK out of beta

This year, Firebase announced a new Crashlytics SDK, which should replace Fabric’s Crashlytics. The new SDK, that just graduated from beta, is more consistent with the other Firebase SDKs and offers improved APIs. It also includes a SDK for NDK and a gradle Plugin. This gradle plugin should replace the previous fabric plugin that was necessary to use the Crashlytics SDK.

What I personally like about this SDK is the fact that it offers a Singleton just like all the other Firebase libraries. This makes it easier to mock a Crashlytics instance when writing tests for you app. So instead of Fabric.with(getContext(), Crashlytics()), you’ll now be using FirebaseCrashlytics.getInstance().

And to learn how to upgrade to the new Firebase Crashlytics SDK, you can check the migration guide.

Atomic field value increments for the Realtime Time Database

This is a feature that I was hoping for. As you might know, it is hard to keep counters or any other field that might suffer concurrent modifications. In order to solve this problem, you can either make use of Cloud Functions or update your fields using Transactions. Starting in version 19.3.0, you can now use ServerValue.increment(double) to increment your counters atomically.

For example, if you were to increment a user’s score in the database, you can simply do:

val reference: DatabaseReference = ...
reference.child("userId").updateChildren(
  hashMapOf("score" to ServerValue.increment(5)) //incrementing by 5
)

Firebase iOS SDK

Obviously, the iOS SDK couldn’t be left behind. Like the Android SDK, it also got:

  • The new Crashlytics SDK out of beta;
  • Realtime Database’s atomic field value increments with ServerValue.increment().

Server Admin SDKs

Node.js

  • Since this SDK depends on the JavaScript client SDK, it has inherited the admin.database.ServerValue.increment() for atomic field value increments.
  • ML Kit for Firebase now provides an API for creating, updating, getting, listing, publishing, unpublishing, and deleting Firebase-hosted custom ML models. You can use this API by calling admin.machineLearning() learn more on the Firebase Documentation.
  • Another API that was recently added is the Remote Config API, which enables developers to programmatically manage the set of JSON-formatted parameters and conditions known as the Remote Config template. The API can be used by admin.remoteConfig() and you can see all the available functions on the Reference Docs.

Python

Just like the Node.js Admin SDK, the Python Admin SDK also got the api to manage hosted ML modules. You can use it by importing the ml module: from firebase_admin import ml. More usage details on the Firebase Documentation.

.NET

The Admin .NET Authentication SDK has 2 new APIs: RevokeRefreshTokensAsync()for revoking any previously issued refresh tokens to a user and VerifyIdTokenAsync() to check if a given ID token has been revoked since it was issued.

Firebase Extensions

storage-resize-images now supports WebP

The storage-resize-images extension was backed by the imagemagick npm module, which does not include WebP support. In order to provide WebP support, the Extensions Team changed the extension to use the sharp npm module instead. This change comes in version 0.1.6 of the extension. See the documentation to find out how to upgrade extensions.

Other Minor Releases

Android

  • Analytics 17.4.0 - bug fixes and deprecation of the Install Referrer Broadcast Receiver;
  • Cloud Firestore 21.4.3 - limited the number of concurrent document lookups it will perform when resolving inconsistencies in the local cache.
  • Cloud Messaging 20.1.6 - Bug fixes
  • In-App-Messaging and In-App-Messaging Display 19.0.6 - bug fixes.
  • Firebase Instance IID 20.1.6 - improved FirebaseOptions validation. This may not sound like a huge change, but it might show you some errors if you do not provide an api key, an app id and a project id. In some cases, re-downloading the google-services.json file from the Firebase Console might help you fix the errors.
  • MLKit Common 22.1.1, Predictions 19.0.7 and Remote Config 19.1.4 - internal changes to later migrate from Firebase Instance IID to Firebase Installations.

iOS

Bug fixes and internal changes on the modules: Analytics, Auth, Cloud Firestore, In App Messaging, FCM and Storage.

JavaScript (Web) 7.14.2

  • Cloud Firestore now rejects write operations if they cannot be persisted in IndexedDB. Previously, these errors crashed the client. This should also fix more crashes related to IndexedDB.
  • Cloud Messaging - changed to compare hostname instead of the entire URL to decide if there is a matching window client.

Unity 6.14.0

  • Cloud Firestore - Firestore.LoggingEnabled is replaced with Firestore.LogLevel for consistency with other Firebase Unity APIs and its getter was removed.
  • Crashlytics - (iOS) Removed references to UIWebView APIs to prevent App Store rejections.

Server Admin SDKs

  • Node. Auth - Exposed email, email_verified, phone_number, and picture fields from the DecodedIdToken type.
  • Go 3.12.1 - Deferred credential loading until required, which allows some APIs like auth.VerifyIDToken() to be called without credentials.
    - Cloud Firestore updated the remote endpoint used by topic management operations.

Firebase Extensions

  • firestore-bigquer-export-v0.1.4 - bug fixes

Cloud Functions

  • Cloud Functions v3.6.1 - updates TypeScript dependency to fix build issues.

Firebase CLI

  • Firebase Tools v8.1.1 - Fixes error when deploying Cloud Functions for Firebase from a project subdirectory.
