Machine Learning — Mantendo diálogos

Para quem acompanhou o último artigo da série Machine Learning, trago este artigo que vai cumprir a promessa que fiz: Mostrar como criar diálogos que variam de acordo com as escolhas do utilizador.

Vamos começar por falar de diálogos. Quando se trata de interações por voz, existem 2 tipos de diálogos:

1. Diálogo Linear

O bot usa o diálogo linear para obter informações necessárias para concluir uma acção. Para criar um diálogo linear, utilizamos um conceito já conhecido: Parâmetros obrigatórios.

Por exemplo, vamos supor que temos um agente de uma loja que vende salgados por encomenda. Para fazer uma encomenda, o nosso utilizador pode falar de várias formas:

“Preciso de rissois”
“Quero rissois com piri-piri”
“Encomenda 1 dúzia de chamussas sem piri piri para amanhã”

E para fazer a encomenda, precisamos de 3 parametros obrigatórios:

  • Nome do salgado
  • Quantidade
  • Com ou sem piri piri

Quando definimos parâmetros como obrigatórios, temos a possibilidade de adicionar “perguntas” (prompts). O bot utiliza estas perguntas como forma de obter os parametros que o utilizador não especificou na requisição:

O agente fará estas perguntas até preencher todos os paramêtros obrigatórios. Por exemplo:

Utilizador — Quero rissois
Bot — Quantas dúzias quer?
Utilizador — Duas
Bot — Com ou sem piri piri?
Utilizador — Sem Piri Piri
Bot — Encomenda registada.

2. Diálogo Não Linear

O diálogo não linear é aquele mencionado no começo do artigo: os diálogos que dependem da resposta dos utilizadores.

Vamos supor que temos um inquérito para avaliar a satisfação dos participantes de um evento. E este inquérito começa com 2 questões:

  • Como avalia o programa do evento?
  • Como avalia o local de realização do evento?

E para cada uma destas questões temos 3 opções:

  • Mau
  • Razoável
  • Bom

Para construir este tipo de diálogo, utilizamos outro conceito já conhecido: Contexts. Começamos por criar um Intent que dá início à conversa. Este Intent terá “avaliar-evento” como Output Context e os Intents seguintes terão isto como Input Context.
